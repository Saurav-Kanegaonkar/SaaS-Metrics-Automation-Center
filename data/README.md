# Data README

All datasets are synthetic and generated by `scripts/score_operating_data.py`. They do not represent real customer or company performance.

## Generation assumptions

- Accounts are distributed across SMB, mid-market, enterprise, and strategic segments.
- ARR, active integration flows, and daily run volume scale by segment.
- Error rate, data quality, dashboard freshness, support tickets, ARR at risk, and expansion signal are generated from related distributions so the metrics move together.
- Five accounts receive a late period incident lift to create realistic anomaly and triage behavior.
- Five accounts receive a product adoption lift to create expansion opportunity signals.
- Metric definitions include certified, draft, and owner gap states to model self serve reporting governance.
- Reporting requests include manual hours, dashboard reuse, data readiness, executive visibility, and AI assist fit so automation readiness can be scored.

## Datasets

- `entities.csv`: account grain with segment, ARR, lifecycle stage, owner team, implementation maturity, and primary system.
- `daily_metrics.csv`: daily integration flow, usage, support, quality, freshness, risk, and expansion signals.
- `source_events.csv`: modeled source exceptions such as schema drift, late extracts, mapping exceptions, and stakeholder requests.
- `metric_definitions.csv`: metric owner, certification, freshness SLA, quality, source table, and blocker fields.
- `reporting_requests.csv`: recurring stakeholder reporting demand and automation readiness inputs.
- `recommended_actions.csv`: account level actions generated from the scored priority queue.
- `analysis/outputs/priority_queue.csv`: account priority scoring output.
- `analysis/outputs/metric_certification_queue.csv`: metric trust workbench output.
- `analysis/outputs/automation_queue.csv`: AI assisted reporting queue output.
